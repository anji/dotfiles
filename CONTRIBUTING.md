# Contributing to Dotfiles

Thank you for considering contributing to this dotfiles configuration!

## How to Contribute

### Reporting Issues

- Check if the issue already exists
- Provide detailed information about your system
- Include steps to reproduce the problem
- Mention which component is affected (Zsh, Neovim, Zellij, etc.)

### Suggesting Enhancements

- Explain why this enhancement would be useful
- Provide examples of how it would work
- Consider backward compatibility

### Pull Requests

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your changes on a clean system if possible
5. Update documentation (README.md) if needed
6. Commit with clear messages
7. Push to your fork
8. Create a Pull Request

## Code Style

### Shell Scripts

- Use 4 spaces for indentation
- Add comments for complex logic
- Follow existing naming conventions
- Test scripts before submitting

### Configuration Files

- Maintain existing format and style
- Comment your additions
- Keep things modular
- Don't break existing functionality

### Documentation

- Use clear and concise language
- Update README.md for new features
- Add inline comments where needed
- Include examples when helpful

## Testing

Before submitting a PR:

1. Test on a fresh system or VM if possible
2. Ensure backup functionality works
3. Verify all components install correctly
4. Check for conflicts with existing configs

## Questions?

Feel free to open an issue for discussion before starting work on major changes.
